# Contract: Task Route

Every task-first route must provide:

| Field | Meaning |
|---|---|
| Situation | Plain-language starting problem |
| Input | What the user supplies |
| Result | The useful work product |
| Boundary | What the software does not decide or where review is required |
| Primary link | A normal HTML link to the live tool |
| Evidence link | A preview or repository route for inspection |

MeetingProof catalogue parity remains governed by `store/app-index.csv`, `store/index.html`, and `store/previews.json`.
