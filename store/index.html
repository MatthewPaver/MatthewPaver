Featured build

Happening: messy venue pages into checked event data.

The hard part was not scraping one website. It was making 103+ different source shapes predictable enough to run, inspect, and trust.

Problem Event listings are scattered, inconsistent, and easy to break.
Decision Treat each venue as a configured source with explicit extraction rules.
Proof 103+ source configs, dedupe, checks, scheduled runs, and a 167-test reliability suite.
Lesson Good automation is not just collection. It is collection you can explain when it fails.
